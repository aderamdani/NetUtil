# Contributing to NetUtil

First off, thank you for considering contributing to NetUtil! It's people like you that make NetUtil such a great tool.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct. Please report unacceptable behavior to the project maintainers.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub issue tracker.
- Describe the bug in detail and provide steps to reproduce it.
- Include your macOS version and NetUtil version.

### Suggesting Enhancements
- Check the issue tracker to see if the enhancement has already been suggested.
- Open a new issue with a clear title and description.

### Pull Requests
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Make your changes.
4. Ensure your code follows the existing style (see `GEMINI.md` for coding standards).
5. Commit your changes (`git commit -m 'Add some amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request.

## Coding Standards
Please refer to **[GEMINI.md](./GEMINI.md)** for detailed coding standards and architectural patterns used in this project. Key points:
- Use **SwiftUI** and **Swift 6**.
- Stick to the **MVVM** pattern.
- No third-party dependencies.
- Use `@MainActor` for UI updates.

## Build Requirements
- macOS 15+
- Xcode 16+
